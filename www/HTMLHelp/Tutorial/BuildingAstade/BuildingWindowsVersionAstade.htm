Getting Astade Model From The SVN site.

If you need to install TortoiseSVN, you can download it from here. You need to create a directory to where TortoiseSVN downloads Astade files. In this example, the directory is named AstadeSVN2. Use right mouse click in the directory to "checkout..." files.  

Copy exact URL as apear in the dialog box.

You will see download status.

When download is finished, the directory should be like this. You should pay attention the directory called "Model" for this tutorial.

Building new Astade with Astade installed

After you open Astade, you should use "open model..." It sounds very obvious that you should select "Model" directory, and it's what should choose.

Astade has all nine components. You should "generate all" the code and compile it. Most of Astade components use "AstadeAPI" library, so you should build it first. Set the component as "active component" before you generate all codes.

Generate all codes for AstadeAPI before you build it.

Make sure that you build the library with Makefile in Windows_StatLib for this tutorial.

If building process is finished, you should repeat the same procedure for rest of eight components. You will get an error lik below when building some components. It's because Makefile require include file from wxWidget, but auto-generated Makefile contains different path.

I guess that it depends on where you install wxWidget. To work around this issus, you need to edit auto-generated Makefile with editor. You should be able to find the marked path from auto-generated Makefile. 

 

You can directly change the marked path to your right path or you can add a variable like below. Of course, the path should be the directory where you find the appropriate include files for wxWidget.

Then you should replace the variable with the path that cause the error. Skip this part if you directly change it with your right path.

When you finished building process of all the nine components, you will find "the executable files" in appropriate directories. You should copy, at least, new Astade.exe file into "C:\Program Files\Astade" in where the old one is.  Now this is very tricky because some components has a cryptic directory name where you keep object files and an executable file.
