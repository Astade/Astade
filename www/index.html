Mission

The goal of this project is, to develop an UML tool where you can develop at model level and get automatic source code for C++.

The program shall  be a complete front-end (GUI) for software development (in C++) which can be customized for any compiler.

At the first look, this seems to be a big deal. But combining some tools, already available, it will be possible!

The main ideas to minimize afford are;

the model tree is build in the directory structure of the operating system.
all graphics are done with graphviz
the documentation is done with doxygen

So we and up with al little "user dialog" and a good "code generator", that should be possible.

Why reinvent the wheel?

After working with an UML tool for over a year now in my job I've learned:

Its great, to work with an UML tool. You'll never want to write source code by hand, after that experience!
Commercial tools are expensive! Really!
Open source tools are not really usable yet :-(
Both, the commercial and the open source ones, are to complicate. You will see that, if you see how simple astade works at the end (look at the software architecture).
